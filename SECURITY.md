# Security

## Lab Safety Statement

This repository is a controlled personal training lab. It is intentionally vulnerable so you can practice AppSec concepts safely.

## Rules

- Do not use real secrets anywhere in the lab.
- Do not scan or attack third-party systems.
- Only run active scanning against the Azure App Service deployed by this repository in your own subscription.
- Treat the app as intentionally vulnerable training code, not production software.

## Responsible Use

- Use the lab to learn how to identify, prioritize, and remediate findings.
- If you add real data, real credentials, or external targets, you are outside the intended safety boundary for this project.
